# Markdown 渲染示例

## 测试用例

你可以在聊天中尝试以下问题，测试 Markdown 渲染功能：

### 1. 基础格式测试

**提示词**：

```
请用 Markdown 格式介绍一下 JavaScript 的基本特性，包括标题、列表和代码示例。
```

**预期效果**：

- 标题层级正确渲染
- 列表格式正确
- 代码块有语法高亮

---

### 2. 代码高亮测试

**提示词**：

```
写一个 Python 快速排序算法，用 Markdown 格式展示，包含注释说明。
```

**预期效果**：

- Python 代码语法高亮
- 注释正确显示
- 缩进保持

---

### 3. 表格测试

**提示词**：

```
用 Markdown 表格比较 React、Vue 和 Angular 三个框架的特点。
```

**预期效果**：

- 表格正确渲染
- 对齐格式正确
- 表格边框样式美观

---

### 4. 链接测试

**提示词**：

```
介绍前端开发的学习路径，并提供相关的学习资源链接（Markdown 格式）。
```

**预期效果**：

- 链接可点击
- 在新标签页打开
- 链接样式突出

---

### 5. 混合格式测试

**提示词**：

```
用 Markdown 格式创建一个 "如何使用 Git" 的教程，包含：
- 标题和子标题
- 命令示例（代码块）
- 步骤列表
- 注意事项（引用块）
```

**预期效果**：

- 所有 Markdown 元素正确渲染
- 格式层次清晰
- 阅读体验良好

## Markdown 支持的元素

### 已支持 ✅

- 标题 (H1-H6)
- 段落
- 粗体 / 斜体
- 行内代码
- 代码块（带语法高亮）
- 有序列表
- 无序列表
- 链接
- 表格
- 引用块
- 水平分割线
- 任务列表（GitHub Flavored Markdown）

### 代码高亮支持的语言

常见编程语言都支持语法高亮，包括：

- JavaScript / TypeScript
- Python
- Java
- C / C++ / C#
- Go
- Rust
- PHP
- Ruby
- Swift
- Kotlin
- HTML / CSS
- SQL
- Bash / Shell
- JSON / YAML
- Markdown
- 等等...

---

## 注意事项

1. **用户消息不渲染 Markdown**：只有 AI 的回复支持 Markdown 渲染，用户输入保持纯文本显示。

2. **代码块建议**：使用三个反引号 (```) 包裹代码块，并指定语言类型以获得最佳高亮效果。

3. **链接安全**：所有链接都会在新标签页打开，并添加了 `rel="noopener noreferrer"` 安全属性。

4. **响应式表格**：表格在移动端会自动适配，必要时可以横向滚动。

---

## 样式定制

Markdown 内容使用 `@tailwindcss/typography` 插件提供的 `prose` 类进行样式化：

- `prose-sm`: 小号字体，适合聊天界面
- `dark:prose-invert`: 深色模式自动反色
- `max-w-none`: 移除最大宽度限制

你可以在 `components/message-item.tsx` 中自定义 Markdown 组件的样式。
